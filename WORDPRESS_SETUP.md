# WordPress版 セットアップガイド

## 0. 前提

このツールは FANZA同人 の商品情報を取得して記事を自動生成し、
WordPress に **下書き（draft）として保存するだけ** です。
**自動で公開は一切行いません。** 公開するかどうか・いつ公開するかは、
必ず自分の目で内容を確認してから、WordPress管理画面で手動判断してください。

---

## 1. レンタルサーバーを契約する（アダルトコンテンツ許可のもの）

一般的な国内レンタルサーバー（エックスサーバー、ロリポップ等）は規約でアダルト
コンテンツを禁止しているところが多いため、**必ず「アダルトコンテンツ許可」を
明記しているサーバーを選んでください。** 例：カラフルボックス（BOX2プランなど）。

契約時のチェックポイント：
- 利用規約に成人向けコンテンツ・アダルトアフィリエイトが明記されているか
- WordPressの簡単インストール機能があるか（あれば作業が大幅に楽になります）
- 独自ドメインを取得・設定できるか

## 2. WordPress本体をインストールする

多くのアダルト対応サーバーには「WordPress簡単セットアップ」的な機能があります。
その場合はサーバーの管理画面（コントロールパネル）からポチポチ進めるだけで、
数分でWordPressサイトが完成します。

手動で入れる場合の大まかな流れ：
1. サーバーにドメインを追加する
2. サーバーのデータベース管理からMySQLデータベースを新規作成する
3. wordpress.org から最新版をダウンロードし、サーバーにアップロード
4. ブラウザでサイトURLにアクセスし、画面の指示に従ってDB接続情報を入力
5. 管理者ユーザー名・パスワードを設定してインストール完了

## 3. 年齢確認・アダルト表記を設定する

- トップページや各記事に「18歳未満はご利用いただけません」等の年齢確認の
  導線を設置する（年齢確認プラグインや、テーマのカスタムHTMLで対応可能）
- アフィリエイトである旨（PR表記）を景品表示法・ステマ規制に沿って明記する
- これらはツールの範囲外で、運用者の責任になります

## 4. アプリケーションパスワードを発行する（重要）

WordPress 5.6以降は標準で「アプリケーションパスワード」機能があり、
通常のログインパスワードを直接使わずにREST API連携ができます。

1. WordPress管理画面 → ユーザー → プロフィール（自分のアカウント）を開く
2. ページ下部の「アプリケーションパスワード」欄までスクロール
3. 新しいアプリケーションパスワード名（例: `github-actions-poster`）を入力し、
   「新しいアプリケーションパスワードを追加」をクリック
4. 表示される文字列（スペース区切りの英数字）をコピーして控える
   → これが `WP_APP_PASSWORD` になります（**この画面を閉じると二度と表示されません**）

もしこの項目が表示されない場合：
- サイトがHTTPS化されていない可能性があります（アプリケーションパスワードは
  HTTPS環境が前提です）。まずSSL証明書を設定してください（多くのアダルト対応
  サーバーでは無料独自SSLが使えます）
- セキュリティプラグイン等で機能が無効化されている場合があります

## 5. GitHub Secretsを登録する

リポジトリの Settings → Secrets and variables → Actions で以下を登録します。

| Secret名 | 内容 |
|---|---|
| `DMM_API_ID` | DMMアフィリエイトAPIのID（既存のものを流用可） |
| `DMM_AFFILIATE_ID` | DMMアフィリエイトID（既存のものを流用可） |
| `WP_URL` | WordPressサイトのURL（例: `https://example.com`、末尾スラッシュなし） |
| `WP_USERNAME` | WordPressのログインユーザー名 |
| `WP_APP_PASSWORD` | 手順4で発行したアプリケーションパスワード |
| `ANTHROPIC_API_KEY` | （任意）記事本文をAI生成したい場合。未設定でもテンプレートで動作します |

## 6. 動作確認

1. GitHub Actions タブ → 「FANZA同人 → WordPress 下書き生成」を選択
2. 「Run workflow」から手動実行（`max_articles` を1〜2件程度に絞ってテスト推奨）
3. 実行後、WordPress管理画面 → 投稿 → 下書き一覧に記事が入っているか確認
4. 記事の内容・画像・リンクを目視確認してから、問題なければ手動で公開する

## 7. 運用開始後の注意

- 現在の設定は **1日2回・最大5件/回** の自動下書き生成です
  （`.github/workflows/wordpress_blog_post.yml` の `cron` と `max_articles` で調整可）
- `age_safety_filter.py` のキーワードは出発点です。実際に取得される作品の
  タイトル・ジャンル表記を見ながら、継続的に拡充してください
- サーバーのアダルトコンテンツ許可条件（転送量上限・深夜アクセス集中への
  対応等）に問題がないか、契約プランを定期的に見直してください
